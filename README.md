# 🔐 Phishing Triage Toolkit

## Overview

The Phishing Triage Toolkit was developed as part of the DecodeLabs Cyber Security Internship Program.

The purpose of this project is to help users identify, analyze, classify, and respond to phishing threats through a structured triage process. The toolkit is designed for both technical and non-technical users and focuses on practical cybersecurity awareness and incident response.

---

## Project Objectives

* Identify phishing attempts.
* Classify suspicious communications.
* Analyze email headers and URLs.
* Apply phishing response procedures.
* Strengthen the Human Firewall approach.

---

## Features

✅ Human Firewall Framework

✅ Email Header Analysis Checklist

✅ URL Inspection Checklist

✅ Phishing Threat Classification

✅ Decision Tree for Threat Analysis

✅ Incident Response Procedures

---

## Threat Categories Covered

* Business Email Compromise (BEC)
* Payment Failure Scams
* Quishing (QR Phishing)
* Deepfake Voice Phishing
* Callback Phishing (TOAD)
* Password Reset Scams
* HR Policy Impersonation
* Fake Meeting Invitations

---

## Human Firewall Framework

### Pause

Stop and think before interacting with suspicious content.

### Verify

Confirm requests through trusted communication channels.

### Report

Report suspicious messages to the appropriate security team.

---

## Classification Levels

### Safe

No suspicious indicators detected.

### Suspicious

Contains unusual characteristics requiring verification.

### Malicious

Contains indicators of phishing, spoofing, credential theft, or fraud.
